## Project Legion — Major Fixes (Evening Session)

### Context
Project Legion was DOWN — 77 jobs stuck in "submitting", 0% success rate. Fixed across 3 Claude Code sessions on the Dell.

### Bug Fixes Applied
1. **Event loop crash** — `asyncio.run()` replaces deprecated `get_event_loop()` in `async_helper.py`
2. **SubmissionDirector wrong method** — `process_approved_job()` not `submit_application()` in `worker.py`
3. **LLM Router 0 providers** — accepts flat config format from `settings.yaml` + fixed config path
4. **Resume module crash** — graceful file loading, cross-platform API key discovery
5. **IT Operations SQLite errors** — replaced with Redis `scard()` queries
6. **Indeed Cloudflare** — Camoufox + DataImpulse proxy for discovery AND submission
7. **trigger_discovery.py** — rewritten for Redis queues (was calling non-existent Mac paths)

### New Feature: External ATS Redirect Handling
**Before:** Indeed jobs that redirect to Workday/Greenhouse/Lever always failed.
**After:** 3-layer retry chain:
- Layer 1: IndeedScraper detects `apply_type: external` at discovery, stores `apply_url`
- Layer 2: IndeedCamoufoxHandler tries `_try_external_form()` with LLM Form Filler in-session
- Layer 3: UnifiedRouter auto-retries with correct ATS handler, stores `apply_url` to Redis

### Mac Mini Deployment
Created all-in-one launcher (`run_legion.py`) that runs Hub + Worker on Mac Mini:
- `setup_mac.sh` — one-command setup (Redis, Python, Playwright, Camoufox)
- `install_service.sh` — launchd auto-start on boot
- `trigger_discovery.py` — manual pipeline trigger via Redis queues

### New Redis Metrics
- `metrics:redirect_submissions` — successful external ATS submissions
- `metrics:redirect_failures` — failed external ATS attempts
- IT pipeline report now shows "External ATS" section
