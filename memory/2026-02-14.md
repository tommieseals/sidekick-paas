## Clawdbot Stability Fix - Scheduled Task Conflict

**Problem:** Clawdbot kept crashing/restarting, terminal window flashing on screen

**Root Cause:**
1. Two competing scheduled tasks: "Clawdbot Gateway" AND "Clawdbot Node"
2. Aggressive restart settings: 999 restarts every 1 minute
3. Built-in watchdog loop + scheduled task restarts = conflict

**Fix Applied:**
```powershell
# Disabled redundant task
Disable-ScheduledTask -TaskName "Clawdbot Node"

# Reduced aggressive restart settings
$task = Get-ScheduledTask -TaskName "Clawdbot Gateway"
$settings = $task.Settings
$settings.RestartCount = 3      # Was 999
$settings.RestartInterval = "PT5M"  # Was PT1M (1 minute)
Set-ScheduledTask -TaskName "Clawdbot Gateway" -Settings $settings
```

**Current Task Status:**
- `Clawdbot Gateway` - Running (main task, keeps Clawdbot alive)
- `Clawdbot Node` - DISABLED (was causing conflicts)
- `ClawdbotKnowledgeSync` - Ready (harmless sync task)

**Files involved:**
- `~/.clawdbot/gateway-watchdog.vbs` - Launches watchdog hidden
- `~/.clawdbot/gateway-watchdog.cmd` - Has built-in restart loop (10s between restarts)
- `~/.clawdbot/gateway.cmd` - Main gateway startup with NODE_OPTIONS=--unhandled-rejections=warn

**If crashes return:** Check `~/.clawdbot/logs/watchdog.log` for patterns
