3.2 Failover Architecture
Implement a heartbeat-based failover system:
	•	Mac Mini (primary) sends a heartbeat ping every 5 minutes to a simple health endpoint on the cloud VM
	•	Cloud VM runs a lightweight watchdog script that monitors the heartbeat
	•	If heartbeat is missing for 15 minutes (3 missed pings), cloud agent activates
	•	Cloud agent sends Telegram alert: "⚠️ Local server unreachable. Cloud failover active."
	•	When Mac Mini comes back online, it resumes as primary and cloud goes back to standby

3.3 Security Requirements
	•	Connect cloud VM to home network exclusively via Tailscale (no open ports on home network)
	•	Store API keys in environment variables or cloud secret managers, never in code
	•	Enable UFW firewall on cloud VM, allow only Tailscale traffic + SSH
	•	Use SSH key authentication only, disable password auth

4.1 Sandboxed Shell Access (just-bash)
just-bash is a verified Vercel Labs project that provides a sandboxed bash environment for AI agents. It runs entirely in-memory with no access to the real filesystem unless explicitly granted.

Installation
npm install -g just-bash

Key Capabilities
	•	File operations (read, write, search) in virtual filesystem
	•	Text processing (grep, sed, awk, sort) for log analysis and data manipulation
	•	SQLite queries via sql.js (WASM-based, fully sandboxed)
	•	Controlled network access via curl with URL allow-lists (disabled by default)

Security Model
	•	No access to real filesystem by default (in-memory only)
	•	No network access by default (must explicitly enable with allow-list)
	•	Protected against infinite loops
	•	No binary execution (pure TypeScript interpretation)

Note: The ChatGPT guide showed incorrect API usage. The correct import is from 'just-bash', and the class is either Bash (simple) or Sandbox (Vercel SDK compatible). See github.com/vercel-labs/just-bash for current API.

Memory Compaction

	•	Use local embedding model (nomic-embed-text via Ollama) for memory embeddings — this costs $0
	•	Implement session-based vector search so flushed memories can be recalled when relevant
	•	Set context pruning with cache-TTL mode to auto-expire old context entries

	•	Increase heartbeat interval to 60 minutes during off-peak hours (10 PM – 7 AM)
	•	Use the smallest local model for heartbeat tasks (Qwen 2.5 3B or similar)

	•	Never use Claude Opus or any paid API model for heartbeat checks

6.2 Filesystem Boundaries
	•	Agents can freely read/write within their designated workspace directories
	•	System directories (/etc, /System, /Library) are READ-ONLY for all agents
	•	Any write outside workspace requires explicit approval via Telegram
	•	Use git for version control on all config files the agent manages

6.3 Dry-Run Mode
For the first 2 weeks of any new automation:
	•	Run in dry-run mode: agent logs what it WOULD do, but takes no action
	•	Review logs daily and correct any problematic decisions
	•	Gradually promote to live mode, starting with lowest-risk tasks
	•	Keep destructive actions (delete, send, modify) in dry-run longer than read-only actions

6.4 Budget Hard Stops
	•	Set hard limits in Anthropic API dashboard and OpenRouter dashboard
	•	Alert at 60% of monthly budget
	•	Warning at 80% of monthly budget
	•	Hard stop at 95% — only DTA can authorize continued spending
	•	If a single task consumes >$5 in tokens, abort and alert immediately
